| Header field                            | Request / Response | What it means                                                                    | Security relevance                                                             |
| --------------------------------------- | -----------------: | -------------------------------------------------------------------------------- | ------------------------------------------------------------------------------ |
| `Host`                                  |            Request | Tells the server which domain the client wants.                                  | Important for virtual hosting. Bad validation can lead to host header attacks. |
| `User-Agent`                            |            Request | NA                         | na.                             |
| `Accept`                                |            Request | Tells the server what content types the client accepts, such as HTML or JSON.    | Low direct security impact.                                                    |
| `Accept-Language`                       |            Request | Tells the server the userâ€™s preferred language.                                  | Can contribute to browser fingerprinting.                                      |
| `Accept-Encoding`                       |            Request | Tells the server which compression methods the client supports, like gzip or br. | Compression can sometimes interact with sensitive data leaks if misused.       |
| `Cookie`                                |            Request | Sends stored cookies back to the server.                                         | Very important. Cookies often contain session IDs.                             |
| `Authorization`                         |            Request | Sends credentials, such as a Bearer token or Basic auth value.                   | Highly sensitive. Should only be sent over HTTPS.                              |
| `Referer`                               |            Request | Shows the page that linked to the current request.                               | Can leak private URLs or tokens if not controlled.                             |
| `Origin`                                |            Request | Shows the origin that made a cross-site request.                                 | Used to defend against CSRF and control CORS.                                  |
| `Content-Type`                          |               Both | Tells what type of data is being sent, such as JSON or HTML.                     | Incorrect values can cause parsing or XSS issues.                              |
| `Content-Length`                        |               Both | Shows the size of the message body in bytes.                                     | Mismatches can cause request smuggling risks.                                  |
| `Set-Cookie`                            |           Response | Tells the browser to store a cookie.                                             | Very important for login/session security.                                     |
| `Cache-Control`                         |               Both | Controls whether content can be cached.                                          | Sensitive pages should usually use `no-store`.                                 |
| `Location`                              |           Response | Tells the browser where to redirect.                                             | Bad use can cause open redirect issues.                                        |
| `Server`                                |           Response | Identifies the web server software.                                              | Can leak version info to attackers.                                            |
| `Strict-Transport-Security`             |           Response | Forces browsers to use HTTPS for the site.                                       | Strong security header. Helps prevent downgrade attacks.                       |
| `Content-Security-Policy`               |           Response | Controls which scripts, images, frames, and resources can load.                  | Very strong defense against XSS and data injection.                            |
| `X-Frame-Options`                       |           Response | Controls whether the page can be loaded inside a frame.                          | Helps prevent clickjacking.                                                    |
| `Frame-Options` / CSP `frame-ancestors` |           Response | Modern way to control who can embed the page.                                    | `frame-ancestors` is preferred in CSP.                                         |
| `X-Content-Type-Options`                |           Response | Usually set to `nosniff`, stopping browsers from guessing file types.            | Helps prevent MIME sniffing attacks.                                           |
| `Referrer-Policy`                       |           Response | Controls how much referrer information the browser sends.                        | Reduces leakage of sensitive URLs.                                             |
| `Permissions-Policy`                    |           Response | Controls browser features like camera, microphone, geolocation, and fullscreen.  | Limits what the page and embedded content can use.                             |
| `Cross-Origin-Resource-Policy`          |           Response | Controls which origins can load the resource.                                    | Helps defend against cross-origin data leaks.                                  |
| `Cross-Origin-Opener-Policy`            |           Response | Isolates the browsing context from other origins.                                | Helps protect against cross-origin attacks.                                    |
| `Cross-Origin-Embedder-Policy`          |           Response | Requires loaded resources to explicitly allow being embedded.                    | Used with COOP for strong browser isolation.                                   |
| `Access-Control-Allow-Origin`           |           Response | Tells browsers which origins may access the response using CORS.                 | Misconfiguration can expose private data. Avoid using `*` with credentials.    |
| `Access-Control-Allow-Credentials`      |           Response | Allows cookies or credentials in cross-origin requests.                          | Risky if combined with weak CORS origin rules.                                 |
| `Access-Control-Allow-Methods`          |           Response | Lists allowed HTTP methods for CORS.                                             | Should only allow needed methods.                                              |
| `Access-Control-Allow-Headers`          |           Response | Lists allowed custom request headers for CORS.                                   | Should be limited to what the app needs.                                       |
| `Access-Control-Max-Age`                |           Response | Tells browser how long to cache CORS preflight results.                          | Mostly performance-related.                                                    |
| `WWW-Authenticate`                      |           Response | Tells the client what authentication method is required.                         | Used for authentication flows.                                                 |
